1. Introducción

La comunicación en redes informáticas se basa en un conjunto de reglas y estándares conocidos como protocolos. Sin ellos, la interoperabilidad entre diferentes fabricantes y sistemas sería imposible. En este análisis, profundizaremos en las arquitecturas que permiten el flujo de datos global.

2. Modelos de Referencia: OSI vs TCP/IP

Para entender cómo viaja la información, dividimos el proceso en capas. El modelo OSI es el estándar teórico, mientras que TCP/IP es la arquitectura real de Internet.

  • Capas del Modelo OSI: Física, Enlace de datos, Red, Transporte, Sesión, Presentación y Aplicación.
  • Capas del Modelo TCP/IP: Acceso a red, Internet, Transporte y Aplicación.

Comparativa Técnica de Modelos

Capa OSI Capa TCP/IP Unidad de Datos (PDU) Función Principal
Aplicación, Presentación, Sesión Aplicación Datos Interfaz de usuario y servicios de red (HTTP, DNS).
Transporte Transporte Segmentos Conexión extremo a extremo y control de flujo.
Red Internet Paquetes Direccionamiento lógico y determinación de rutas (IP).
Enlace de Datos Acceso a Red Tramas Direccionamiento físico (MAC) y detección de errores.
Física Acceso a Red Bits Transmisión binaria por medios físicos (cables, ondas).

Proceso de Encapsulamiento

A medida que los datos descienden por las capas, se les añade información de control (headers). Al llegar a la Capa de Transporte se denominan Segmentos, en la Capa de Red son Paquetes, y en la de Enlace son Tramas.

3. Modelos de Comunicación

Existen dos formas principales de organizar la interacción entre nodos:

  • Cliente/Servidor: Un host central (servidor) ofrece recursos y los clientes los consumen. Ejemplo: Un servidor web.
  • De Par a Par (P2P): No hay jerarquía; cada computadora puede actuar como cliente y servidor al mismo tiempo, facilitando el intercambio de archivos directamente.

4. Conflictos de Red y Segmentación

Para asegurar un flujo confiable, debemos gestionar los siguientes dominios:

Dominio de Colisión

Es el segmento donde dos dispositivos pueden transmitir al mismo tiempo provocando un choque de señales. Los Switches solucionan esto creando microsegmentos por cada puerto.

Dominio de Broadcast

Es el área donde un mensaje de difusión llega a todos los nodos. Demasiado tráfico de este tipo ralentiza la red. Los Routers se utilizan para limitar estos dominios.

5. Conclusión

El dominio de los protocolos de la Unidad V es vital para cualquier informático. La correcta segmentación de la red y el entendimiento de los modelos OSI/TCP/IP permiten diseñar infraestructuras robustas, escalables y eficientes, minimizando errores y maximizando la velocidad de transferencia.
